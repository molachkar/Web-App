building order: 

1. core/config.py          ← no dependencies
2. core/cache.py           ← depends on config
3. data/fred.py            ← depends on config
4. data/prices.py          ← depends on config
5. data/candle_validator.py← depends on config
6. features/engineer.py    ← depends on config
7. ml/inference.py         ← depends on features, config
8. smc/engine.py           ← depends on config
9. market/ranges.py        ← depends on prices
10. dom/handler.py         ← depends on config
11. core/scheduler.py      ← depends on everything above
12. server.py              ← imports all, wires routes
13. frontend/index.html    ← built last, calls clean JSON endpoints

calling logic: 

@app.get("/signal")      → data pipeline → ml inference → return JSON
@app.get("/smc")         → fetch 4H → smc engine → return JSON  
@app.get("/prices")      → price strip → return JSON
@app.get("/candles")     → mt5 candles → return JSON
@app.get("/status")      → candle validator + safe window → return JSON
@app.websocket("/ws")    → DOM broadcast
@app.get("/")            → serve index.html
